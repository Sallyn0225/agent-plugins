# Code of Conduct

## Our Standards

We commit to a welcoming, respectful, and harassment-free community. Positive behavior includes empathy, constructive technical feedback, respect for differing experience and viewpoints, graceful acceptance of correction, and focus on what is best for users and contributors.

## Unacceptable Behavior

Unacceptable behavior includes harassment; discrimination; sexualized language or attention; insults, threats, trolling, or sustained disruption; publishing another person's private information; exposing credentials or undisclosed vulnerabilities; and other conduct that would reasonably be considered inappropriate in a professional setting.

## Scope

This Code applies in repository issues, pull requests, reviews, discussions, release channels, and other project spaces. It also applies when a person officially represents the project in public.

## Reporting and Enforcement

Report conduct concerns privately to the repository maintainers through an appropriate private GitHub channel. Security vulnerabilities belong in [GitHub Private Vulnerability Reporting](SECURITY.md#reporting-a-vulnerability), not conduct reports.

Maintainers will review reports fairly, protect reporter privacy as far as practical, and may remove content, issue a warning, temporarily restrict participation, or permanently ban participation according to severity and context. Maintainers must recuse themselves when they have a conflict of interest.
